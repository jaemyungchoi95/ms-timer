# ms-timer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 18:00:00.000까지 남은 시간을 1ms 단위로 표시하는 Windows 데스크톱 앱(portable exe)을 만든다.

**Architecture:** 시간 계산은 `Date`를 주입받는 순수 함수 하나(`computeRemaining`)로 격리하여 node:test로 전 경계를 검증한다. Electron main은 BrowserWindow 생성만 담당하고, renderer는 `requestAnimationFrame` 루프에서 매 프레임 절대 시각 차분(`todayAt18() - now`)을 재계산해 DOM을 갱신한다. IPC·preload는 없다 — renderer가 필요한 것은 `Date.now()`뿐이다.

**Tech Stack:** Electron 43, electron-builder 26, node:test (내장), 순수 ESM (빌드 스텝 없음, 런타임 의존성 0)

**Spec:** `docs/superpowers/specs/2026-07-15-ms-timer-design.md`

- [ ] **Step 2: 실패하는 테스트 작성**

`at()` 헬퍼는 **로컬 시각** Date를 만든다(월은 0-indexed이므로 `6` = 7월). `computeRemaining`의 `setHours`도 로컬 기준이라 타임존과 무관하게 일관된다.

마지막 두 테스트가 중요하다: `targetHour` 파라미터화는 spec의 "필요 시 확장 가능"을 보장하고, 순수성 테스트는 `now.setHours()`로 입력을 오염시키는 실수를 잡는다.

```js
import { test } from 'node:test';
import assert from 'node:assert/strict';
import { computeRemaining } from '../src/lib/countdown.js';

const at = (h, m, s, ms) => new Date(2026, 6, 15, h, m, s, ms);

test('정상 카운트다운 — 09:00이면 9시간 남음', () => {
  assert.deepEqual(computeRemaining(at(9, 0, 0, 0)),
    { expired: false, h: 9, m: 0, s: 0, ms: 0 });
});

test('18시 1ms 전', () => {
  assert.deepEqual(computeRemaining(at(17, 59, 59, 999)),
    { expired: false, h: 0, m: 0, s: 0, ms: 1 });
});

test('18시 정각 — expired', () => {
  assert.deepEqual(computeRemaining(at(18, 0, 0, 0)),
    { expired: true, h: 0, m: 0, s: 0, ms: 0 });
});

test('18시 1ms 후 — expired', () => {
  assert.deepEqual(computeRemaining(at(18, 0, 0, 1)),
    { expired: true, h: 0, m: 0, s: 0, ms: 0 });
});

test('18시 이후 실행 — 즉시 expired', () => {
  assert.deepEqual(computeRemaining(at(19, 30, 0, 0)),
    { expired: true, h: 0, m: 0, s: 0, ms: 0 });
});

test('자정 직후 — 같은 날 18시까지 18시간 (자정 롤오버)', () => {
  assert.deepEqual(computeRemaining(at(0, 0, 0, 0)),
    { expired: false, h: 18, m: 0, s: 0, ms: 0 });
});

test('자릿수 혼합 — 08:07:06.005', () => {
  assert.deepEqual(computeRemaining(at(8, 7, 6, 5)),
    { expired: false, h: 9, m: 52, s: 53, ms: 995 });
});

test('targetHour 인자로 목표 시각 변경', () => {
  assert.deepEqual(computeRemaining(at(17, 0, 0, 0), 19),
    { expired: false, h: 2, m: 0, s: 0, ms: 0 });
});

test('입력 Date를 변경하지 않는다 (순수 함수)', () => {
  const now = at(9, 0, 0, 0);
  const before = now.getTime();
  computeRemaining(now);
  assert.equal(now.getTime(), before);
});
```

- [ ] **Step 4: 최소 구현**

`new Date(now)`는 복사 생성자다 — 입력을 오염시키지 않는다. `now.setHours(...)`를 직접 호출하면 순수성 테스트가 깨진다.

```js
/**
 * 같은 날짜의 목표 시각까지 남은 시간을 계산한다.
 *
 * 절대 시각 차분 방식이므로 드리프트가 누적되지 않고,
 * 절전 복귀 / NTP 보정 / 타임존 변경이 자동으로 반영된다.
 *
 * @param {Date} now 현재 시각 (변경되지 않음)
 * @param {number} targetHour 목표 시(로컬), 기본 18
 * @returns {{expired: boolean, h: number, m: number, s: number, ms: number}}
 */
export function computeRemaining(now, targetHour = 18) {
  const target = new Date(now);
  target.setHours(targetHour, 0, 0, 0);

  const diff = target.getTime() - now.getTime();

  if (diff <= 0) {
    return { expired: true, h: 0, m: 0, s: 0, ms: 0 };
  }

  return {
    expired: false,
    h: Math.floor(diff / 3600000),
    m: Math.floor(diff / 60000) % 60,
    s: Math.floor(diff / 1000) % 60,
    ms: diff % 1000,
  };
}
```

- [ ] **Step 5: clock.js 작성**

`setInterval(1ms)`를 쓰지 않는 이유: 브라우저가 clamp하며, rAF보다 자주 그려도 모니터가 표시하지 못한다.
`lastExpired` 캐싱은 마이크로 최적화가 아니다 — 없으면 `classList.toggle`과 label 텍스트 쓰기가 초당 60회 무의미하게 반복된다. 반면 hms/ms는 매 프레임 값이 실제로 바뀌므로 캐싱하지 않는다.

```js
import { computeRemaining } from '../lib/countdown.js';

const clockEl = document.getElementById('clock');
const hmsEl = document.getElementById('hms');
const msEl = document.getElementById('ms');
const labelEl = document.getElementById('label');

const pad = (n, width) => String(n).padStart(width, '0');

let lastExpired = null;

function tick() {
  const { expired, h, m, s, ms } = computeRemaining(new Date());

  hmsEl.textContent = `${pad(h, 2)}:${pad(m, 2)}:${pad(s, 2)}`;
  msEl.textContent = pad(ms, 3);

  if (expired !== lastExpired) {
    clockEl.classList.toggle('expired', expired);
    labelEl.textContent = expired ? '퇴근' : '퇴근까지';
    lastExpired = expired;
  }

  requestAnimationFrame(tick);
}

requestAnimationFrame(tick);
```

# ms-timer

18시 정각까지 남은 시간을 1ms 단위로 표시하는 Windows 데스크톱 앱.

## 실행

`dist/ms-timer.exe` — 설치 불필요.

18시에 도달하면 `00:00:00.000`에서 멈추고 "퇴근"을 표시한다.
자정을 넘기면 자동으로 다음 날 18시 카운트다운이 시작되므로 켜둔 채로 둬도 된다.

## 왜 ms 자리가 뭉개져 보이나

모니터 refresh rate는 60Hz(16.7ms)다. ms 3자리는 물리적으로 개별 프레임을 볼 수 없으므로
blur 잔상으로 보인다. 이것이 의도된 동작이며, 시간의 흐름을 체감시키는 것이 이 앱의 목적이다.

## 개발

프로젝트는 반드시 `/mnt/c` 아래에 둔다. WSL 파일시스템에 두면 Windows node가
`\\wsl$` UNC 경로로 접근하며 electron-builder가 실패한다.

```bash
node --test                              # 테스트 (WSL node)
cmd.exe /c "npm install"                 # 의존성 (Windows node 필수)
cmd.exe /c "npx electron ."              # 개발 실행
cmd.exe /c "npx electron-builder --win portable"   # exe 빌드 → dist/ms-timer.exe
```

`npm install`을 WSL node로 하면 linux용 electron 바이너리가 받아져 실행되지 않는다.

## 구조

- `src/lib/countdown.js` — 시간 계산 순수 함수. 유일한 로직이자 유일한 테스트 대상
- `src/main.js` — BrowserWindow 생성
- `src/renderer/` — rAF 루프 + DOM 갱신

시간 계산은 절대 시각 차분(`오늘 18시 - now`)이다. 누적 감산이 아니므로
드리프트가 없고 절전 복귀·NTP 보정·타임존 변경이 자동으로 반영된다.
